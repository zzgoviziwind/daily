## 核心概念

### 处理流程分两个阶段

**匹配阶段**（`fillConclusions`）：
遍历体检人的所有结论，根据规则类型决定是否将结论加入对应科室的 `hitConclusions`

**处理阶段**（`getEffectiveConclusions` / `getNeedDeleteConclusions`）：
根据各类型逻辑决定保留哪些结论、删除哪些结论

### 匹配时使用的词（三种来源）

1. `standardName`（标准词）
2. `standardAbnormalTerm`（标准异常词）
3. 同义词表中的父级词（`directParentWord`，addType=1）

---

## Type 1 — 取优先级最高科室的结果，若无异常则结果为无异常

### 匹配条件

- `conflictStandardTerm`（全词列表）优先判断
- 若列表为空，则用 `conflictStandardTermPattern`（正则）匹配

### 处理逻辑

1. 筛选 `hasDone=true` 的科室（已检查的科室）
2. 按 priority 升序排序（数字越小优先级越高）
3. 取第0个（最高优先级）科室的 hitConclusions 作为**有效结论**
4. 其余科室的 hitConclusions 全部**删除**

### 关键：最高优先级科室无匹配结论时

最高优先级科室的 `hitConclusions` 为空 → `getHighestPriorityConclusions()` 返回空列表
→ 有效结论为空，其他科室的结论也全部被删除

---

### 示例

**示例1**：正常场景，超声科和CT科都有结论

```
规则：肾囊肿，Type 1
优先级：超声科(1) > CT科(2) > 内科(3)

超声科结论：肾囊肿
CT科结论：肾囊肿
内科结论：肾囊肿

处理：
  最高优先级 = 超声科，其 hitConclusions = [肾囊肿]
  有效结论 = 超声科的 [肾囊肿]
  删除：CT科的肾囊肿、内科的肾囊肿

最终：只保留超声科的 肾囊肿
```

**示例2**：最高优先级无匹配结论，次级有结论

```
规则：肾囊肿，Type 1
优先级：超声科(1) > CT科(2)

超声科结论：（无肾囊肿相关结论，如只有"高血压"）
CT科结论：肾囊肿

处理：
  超声科 hitConclusions = []（空，未匹配到肾囊肿）
  最高优先级科室无结论 → 有效结论 = 空
  删除：CT科的肾囊肿

最终：肾囊肿结论被全部删除！
```

**示例3**：最高优先级科室根本没有参与体检

```
规则：肾囊肿，Type 1
优先级：超声科(1) > CT科(2)

本次体检未做超声（hasDone=false），CT科有"肾囊肿"

处理：
  hasDone=true 的科室中，CT科最高优先级
  有效结论 = CT科的 [肾囊肿]
  没有其他科室需要删除

最终：保留CT科的 肾囊肿
```

## Type 2 — 科室优先级高到低，找到有结论的即停

### 匹配条件

与 Type 1 相同：`conflictStandardTerm` 列表或 `conflictStandardTermPattern` 正则

### 处理逻辑

1. 按 priority 顺序遍历科室列表
2. 找到第一个 `hitConclusions` 不为空的科室，取其结论为**有效结论**，停止遍历
3. 跳过所有没有匹配结论的高优先级科室
4. 其余科室的匹配结论全部**删除**

### 与 Type 1 的核心区别

| 对比项 | Type 1 | Type 2 |
|--------|--------|--------|
| 最高优先级无结论时 | 全部删除 | 继续向下找 |
| 取的是 | 最高优先级科室 | 第一个有结论的科室 |

---

### 示例

**示例1**：超声科有结论

```
规则：脂肪肝，Type 2
优先级：超声科(1) > CT科(2) > 内科(3)

超声科：脂肪肝
CT科：脂肪肝
内科：脂肪肝

处理：
  遍历到超声科，hitConclusions=[脂肪肝]，停止
  有效结论 = 超声科的 [脂肪肝]
  删除：CT科、内科的脂肪肝

最终：只保留超声科的 脂肪肝
```

**示例2**：超声科无结论，CT科有结论

```
超声科：（无脂肪肝）
CT科：脂肪肝
内科：脂肪肝

处理：
  遍历超声科，hitConclusions=[]，跳过
  遍历CT科，hitConclusions=[脂肪肝]，停止
  有效结论 = CT科的 [脂肪肝]
  删除：内科的脂肪肝

最终：保留CT科的 脂肪肝
```

**示例3**：只有内科有结论

```
超声科：（无脂肪肝）
CT科：（无脂肪肝）
内科：脂肪肝

处理：
  跳过超声科、CT科
  内科有结论，停止
  有效结论 = 内科的 [脂肪肝]

最终：保留内科的 脂肪肝
```

**示例4**：所有科室均无结论

```
所有科室无脂肪肝

处理：
  遍历完毕，没有找到有结论的科室
  有效结论 = 空

最终：没有任何结论被保留（也没有删除）
```

## Type 3 — 取所有参与科室中优先级最高的那批结论

### 匹配条件

**不通过词匹配！**
结论所属科室ID在 `conflictDepartIds` 列表中即可，不关心是什么异常词

### 处理逻辑

1. 将 `conflictDepartIds` 中科室的所有结论都加入对应科室的 `hitConclusions`
2. 筛选 `hasDone=true` 的科室
3. 按 priority 分组排序
4. 只有一个优先级时，不做任何处理（返回空，不删除）
5. 存在多个优先级时，保留最高优先级组的全部结论，删除其余组的结论

### 与 Type 1 的区别

| 对比项 | Type 1 | Type 3 |
|--------|--------|--------|
| 匹配条件 | 标准词/异常词匹配 | 科室ID匹配 |
| 适用场景 | 针对特定疾病 | 针对特定科室的所有结论 |
| 只有一个科室有结论时 | 无需处理 | 无需处理（返回空） |

---

### 示例

**示例1**：两个优先级科室都有结论

```
规则：Type 3
conflictDepartIds = [超声科ID, CT科ID]
优先级：超声科(1) > CT科(2)

超声科结论：肾囊肿、胆囊结石（科室在conflictDepartIds中，全部加入）
CT科结论：肾结石（科室在conflictDepartIds中，全部加入）

处理：
  hasDone科室：超声科(priority=1)、CT科(priority=2) → 两个不同优先级
  保留：超声科的全部结论（肾囊肿、胆囊结石）
  删除：CT科的肾结石

最终：保留超声科的 肾囊肿、胆囊结石
```

**示例2**：只有一个科室做了检查

```
conflictDepartIds = [超声科ID, CT科ID]
优先级：超声科(1) > CT科(2)

超声科：未做体检（hasDone=false）
CT科结论：肾结石

处理：
  hasDone=true 的只有CT科 → 只有一个优先级
  只有一个优先级时，不做任何处理，返回空

最终：CT科的肾结石原样保留（不删除）
```

**示例3**：两个科室同优先级（同时配置了 priority=1）

```
超声科A(priority=1)、超声科B(priority=1) → 同一优先级分组
CT科(priority=2)

处理：
  priority=1 组：超声科A + 超声科B 的结论全部保留
  priority=2 组：CT科的结论全部删除
```

## Type 4 — 最高优先级为主，其余科室做互补（去重合并）

### 匹配条件

只支持 `conflictStandardTermPattern`（正则匹配），不支持全词列表

### 处理逻辑

1. 取最高优先级科室的所有 hitConclusions
2. 提取最高优先级科室结论的 `standardAbnormalTerm` 和 `standardName` 作为去重集合
3. 遍历其他科室的结论，若其 `standardAbnormalTerm` 和 `standardName` **不在**集合中，则保留
4. 最终：最高优先级科室全保留 + 其他科室中不重复的结论也保留

---

### 示例

**示例（代码注释原示例）**：

```
规则：Type 4，正则匹配 "胆囊.*"
优先级：彩超室(1) > 放射科CT(2)

彩超室结论：胆囊结石伴壁毛糙、胆囊息肉样病变
放射科CT结论：胆囊结石、胆囊炎、胆囊息肉样病变

处理：
  最高优先级去重集合 = {胆囊结石伴壁毛糙, 胆囊息肉样病变}
  放射科CT逐一判断：
    - 胆囊结石 → 不在集合中 → 保留
    - 胆囊炎 → 不在集合中 → 保留
    - 胆囊息肉样病变 → 在集合中 → 删除

最终：
  彩超室：胆囊结石伴壁毛糙、胆囊息肉样病变（全保留）
  放射科CT：胆囊结石、胆囊炎（保留互补部分）
```

## 总结对比

| 类型 | 匹配方式 | 匹配字段 | 核心逻辑 | 最高优先级无结论时 | 只有一个科室有结论时 |
|------|----------|----------|----------|--------------------|----------------------|
| **Type 1** | 全词 / 正则 | `conflictStandardTerm` / `conflictStandardTermPattern` | 只取最高优先级科室 | **全部删除** | 不处理 |
| **Type 2** | 全词 / 正则 | `conflictStandardTerm` / `conflictStandardTermPattern` | 取第一个有结论的科室 | 向下找下一个 | 不处理 |
| **Type 3** | 科室ID | `conflictDepartIds` | 只取最高优先级科室 | **全部删除** | 不处理（返回空） |
| **Type 4** | 正则 | `conflictStandardTermPattern` | 最高优先级 + 其他互补 | 保留其他科室不重复的 | 正常保留 |
